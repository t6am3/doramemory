## 配置文件规范

```yaml
# ~/.doramemory/config.yaml

watch:
  - path: ~/.claude/projects/
    format: claude
    memory_file: ~/.claude/CLAUDE.md      # 含 {{DORAMEMORY}} 占位符
  - path: ~/.cursor/conversations/
    format: openai
    memory_file: ~/.cursor/.cursorrules

compression:
  model:
    provider: anthropic                   # anthropic / openai / custom
    model_id: claude-haiku-4-5
    # 自定义时填写：
    # base_url: https://your-endpoint/v1
    # api_key:  sk-...
    # model_id: your-model-name

  embedding:
    provider: local                       # local / openai / custom
    model_id: nomic-embed-text            # local 时通过 ollama 运行
    # 自定义时填写：
    # base_url: https://your-endpoint/v1
    # api_key:  sk-...

cold_start_days: 7
session_gap_minutes: 30                  # 多少分钟无消息视为 session 结束
memory_update_throttle_seconds: 300      # MEMORY.md 最小更新间隔
```

### LLM 配置的三种形态

**Anthropic 标准**：
```yaml
compression:
  model:
    provider: anthropic
    model_id: claude-haiku-4-5
    api_key: ${ANTHROPIC_API_KEY}         # 支持环境变量引用
```

**OpenAI 标准**：
```yaml
compression:
  model:
    provider: openai
    model_id: gpt-4o-mini
    api_key: ${OPENAI_API_KEY}
```

**自定义（任何兼容 OpenAI 接口的服务）**：
```yaml
compression:
  model:
    provider: custom
    model_id: qwen2.5-72b
    base_url: https://dashscope.aliyuncs.com/compatible-mode/v1
    api_key: ${DASHSCOPE_API_KEY}
```

本地模型（ollama）也走 custom，base_url 填 `http://localhost:11434/v1`。

## MEMORY.md 占位符机制

用户在 memory 文件里放置占位符，DoraMemory 负责替换这一段内容：

```markdown
# 用户自己写的项目说明...

## Memory
{{DORAMEMORY}}

# 用户自己写的其他内容...
```

DoraMemory 将 `{{DORAMEMORY}}` 替换为完整的三层结构：

```markdown
## Memory
<!-- DORAMEMORY:START -->
### 我是谁
在 AI agent 记忆领域工作，擅长从混乱中找到系统性解法。

### 重要记忆
DoraMemory 项目启动，这是一次重要的方向转变。

### 近期记忆
=== 昨天 ===
完成 v0.6 设计，确定 recall 排序机制。

=== 上周 ===
项目启动，确定时间维度 Pooling 核心思路。
<!-- DORAMEMORY:END -->
```

**规则**：
- 只替换 `{{DORAMEMORY}}` 或 `DORAMEMORY:START/END` 区块之间的内容
- 占位符不存在时不写入（不强制修改用户文件）
- 用户移动占位符位置，下次刷新自动跟随

## Daemon 管理

```bash
npx doramemory init      # 初始化并启动
npx doramemory status    # 查看运行状态和统计
npx doramemory stop      # 停止
npx doramemory restart   # 重启
npx doramemory backfill --days 30   # 补录历史
npx doramemory uninstall            # 卸载（默认保留数据）
```

**launchd plist（macOS）**：
```xml
<!-- ~/Library/LaunchAgents/dev.doramemory.plist -->
<key>RunAtLoad</key><true/>
<key>KeepAlive</key><true/>
```

daemon 内部集成 FSWatcher + cron scheduler，用户只管一个进程。

## MCP 注册

自动写入 `~/.claude/claude_desktop_config.json`：

```json
{
  "mcpServers": {
    "doramemory": {
      "command": "npx",
      "args": ["doramemory", "mcp"]
    }
  }
}
```

暴露两个 MCP tool：

```
memory_recall    查询历史记忆（语义 or 时间范围）
memory_remember  标记重要记忆或修正摘要
```

## Decision Log

### DDR-025: MEMORY.md 使用占位符而非自动追加
**决策**: 用户在 memory 文件中手动放置 `{{DORAMEMORY}}`，DoraMemory 替换该位置内容。  
**理由**: 用户掌控记忆在 prompt 中的位置，透明可审计，不强制修改用户文件结构。  
**日期**: 2026-04-08  
**状态**: Accepted

### DDR-026: LLM 配置支持三种形态
**决策**: Anthropic / OpenAI 标准接口 + custom（任何兼容 OpenAI 接口的服务），均支持环境变量引用 key。  
**理由**: 用户使用的模型多样，本地 ollama、云端 API、企业内部部署都要能用；OpenAI 接口已成事实标准。  
**日期**: 2026-04-08  
**状态**: Accepted

### DDR-027: Daemon 内置 FSWatcher + cron，不单独注入 crontab
**决策**: 用户只管理一个 daemon 进程，FSWatch 和压缩 cron 都在内部。  
**理由**: 单独注入 crontab 对用户不透明，难以管理；daemon 统一管理更简洁，重启即恢复。  
**日期**: 2026-04-08  
**状态**: Accepted
