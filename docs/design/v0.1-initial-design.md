## 核心思想

人类记忆对近期事件保真度高，对远期事件只保留骨架。将这一特性建模为**时间维度上的分层 Pooling**：

- 每一层对应一个时间粒度（日/周/月/年）
- 层与层之间的转换由**压缩算子**完成（LLM 调用，带认知偏好）
- Agent 站在"现在"这个时间点，看到的记忆分辨率随时间距离递减

## 记忆分层结构

### Agent 当前时刻的记忆全景（分辨率视图）

```
← 过去                                                    现在 ●
──────────────────────────────────────────────────────────────
2年前      1年前      半年前    2月前   上月   上周  昨天  今天
  │          │          │         │       │      │     │     │
  
  覆盖跨度:  [  1年  ]  [ 6月  ]  [ 1月 ] [ 1周] [1天][1天][当下]
  
  ░░░        ▒▒▒▒       ▒▒▒▒▒    ▓▓▓▓▓  ▓▓▓▓▓  ████ ████  ██████
  极稀疏     稀疏        中等      较密    较密    密   全量   Working
  年骨架     月摘要      月摘要    周摘    日摘    日摘  原始   Memory

  resolution:  ~3%       ~8%      ~15%   ~30%  ~50%  ~70%  ~95%  100%
```

### 五层层级定义

| 层级 | 名称 | 时间跨度 | 分辨率 | 存储单元 |
|------|------|----------|--------|----------|
| L0 | Working Memory | 当前会话 | 100% | 完整 context |
| L1 | 日原始片段 | 0~24h | ~95% | 每条事件（时间戳 + 内容 + 情绪标注） |
| L2 | 日摘要 | 1~7 days | ~70% | 每天一篇结构化"日记" |
| L3 | 周摘要 | 1~4 weeks | ~30% | 每周主题 + 决策 + 实体变化 |
| L4 | 月摘要 | 1~12 months | ~10% | 每月大事 + 认知更新 |
| L5 | 年骨架 | >1 year | ~3% | 每年 narrative arc |

## 压缩算子设计

每层压缩不是简单的 summarize，而是携带特定**认知偏好**的 LLM 调用。

### 算子清单

| 算子 | 功能 | 认知类比 |
|------|------|----------|
| **Salience Filter** | 过滤平凡事件，标记显著事件 | 海马体筛选机制 |
| **Pattern Extractor** | 发现重复出现的行为/思维模式 | 直觉形成 |
| **Entity Tracker** | 追踪重要实体（人/项目/概念）的状态变化 | 人际关系记忆 |
| **Belief Updater** | 识别导致认知改变的关键事件 | 经验积累 |
| **Narrative Weaver** | 把碎片编织成有因果关系的故事 | 自传式记忆 |

### 各层算子组合

| 压缩层 | 使用算子 | 说明 |
|--------|----------|------|
| L1→L2（日压缩） | Salience Filter + Entity Tracker | 提炼今天什么重要、实体状态如何变化 |
| L2→L3（周压缩） | Pattern Extractor + Entity Tracker | 发现本周模式，追踪关键实体 |
| L3→L4（月压缩） | Belief Updater + Narrative Weaver | 识别认知更新，编织月度叙事 |
| L4→L5（年压缩） | Narrative Weaver | 只保留 narrative 骨架 |

## 检索机制

### 检索评分公式

```
score(m, q, T) = α · sem(m, q)          # 语义相关性
              + β · decay(T - t_m)       # 时间衰减（T=现在，t_m=记忆时间）
              + γ · salience(m)          # 事件重要性
```

其中 `decay` 函数参考 Ebbinghaus 遗忘曲线：`decay(Δt) = e^(-Δt/λ)`，λ 为衰减常数。

### 分层检索策略

```
Query + 当前时间 T
  │
  ├─ 近3天 → 直接扫描 L1 原始片段
  ├─ 3天~1月 → 搜索 L2/L3 摘要
  ├─ 1月~1年 → 搜索 L4 月摘要
  └─ >1年 → 搜索 L5 年骨架（不可 unfold）
  
→ 各层 Top-K 加权合并 → 注入 context
```

## Flashbulb Memory（闪光灯记忆）

高 salience 事件（分数超过阈值）**抵抗压缩**，永远保留在 L1。

触发条件（示例）：
- salience score ≥ 9/10
- 被 Agent 显式标记为"重要"
- 被多次检索引用（热点记忆）

## 压缩触发时机

| 触发方式 | 适用层级 | 说明 |
|----------|----------|------|
| 会话结束触发 | L1→L2 | 每次对话结束后压缩当天片段 |
| 定时触发（cron） | L2→L3 | 每周一凌晨压缩上周日摘要 |
| 定时触发（cron） | L3→L4 | 每月1日压缩上月周摘要 |
| 定时触发（cron） | L4→L5 | 每年1月1日压缩上年月摘要 |

## Decision Log

### DDR-001: 压缩触发时机
**决策**: 压缩在时间边界触发（日/周/月/年），而非事件数量触发。  
**理由**: 更接近人类记忆的"睡眠巩固"机制；边界明确，易于工程实现。  
**日期**: 2026-04-07  
**状态**: Proposed

### DDR-002: Salience 打分时机
**决策**: 事件写入时打初始分，压缩时回顾修正（双阶段）。  
**理由**: 事件刚发生时难以判断重要性（局部信息）；压缩时有全天视角，更准确。  
**日期**: 2026-04-07  
**状态**: Proposed

### DDR-003: Flashbulb Memory 机制
**决策**: salience ≥ 阈值的事件永久保留在 L1，不参与压缩。  
**理由**: 人类对极端事件的记忆不随时间衰退；agent 也需要"永不忘记"的能力。  
**日期**: 2026-04-07  
**状态**: Proposed
