# DoraMemory 设计方案 v0.2

**版本**: v0.2  
**日期**: 2026-04-07  
**状态**: 草稿（讨论中）  
**变更摘要**: 确定接口层设计、写入触发机制、注入模型、fidelity 衰减公式

---

## 相对 v0.1 的核心变更

| 主题 | v0.1 | v0.2 |
|------|------|------|
| 写入触发 | proxy 拦截 / 被动 hook | compress endpoint，框架主动调用 |
| 记忆注入 | 按语义相关性召回注入 | 纯时间驱动，按天分段，session 开始一次 |
| recall | 系统自动注入 | agent 主动发起，语义检索 |
| fidelity | 定性描述 | 形式化衰减公式 |
| 接口 | 未定义 | 四个明确接口 |

---

## 设计原则（新增）

> **DoraMemory 是数据库 + 检索引擎，不是中间件。**
> 
> 不修改用户原始 prompt，不拦截 LLM 调用链。所有读写均为显式调用。

---

## 四个接口

### `POST /compress`

**调用方**: agent 框架（context 满时自动触发）  
**作用**: 压缩旧消息，同时将原始内容写入 L1 记忆存储

```typescript
Request: {
  session_id: string
  messages:   Message[]   // 需要被压缩的旧消息
  target_tokens: number   // 期望压缩后的 token 数
}

Response: {
  compressed_messages: Message[]  // 返回给框架继续用
  memory_id:  string
  layer:      'L1'
  fidelity:   1.0          // 刚写入，永远是 1
}
```

**关键点**: 写入的始终是 L1，fidelity 永远为 1.0。L1→L2→... 的压缩由 DoraMemory 内部 cron 完成，不在此接口发生。

---

### `GET /memory`

**调用方**: agent 框架（session 开始时调一次）  
**作用**: 返回近期记忆，按天分段，注入 system prompt

```typescript
Request: {
  session_id: string
  days?:       number   // 近几天，默认 5
  max_tokens?: number   // budget，默认 600
}

Response: {
  segments: [
    {
      date:     '2026-04-06',
      label:    '昨天',
      layer:    'L2',
      fidelity: 0.88,
      content:  '上午：与团队确定 compress endpoint 作为写入触发点。下午：完成接口设计。'
    },
    {
      date:     '2026-04-05',
      layer:    'L2',
      fidelity: 0.88,
      content:  '建立仓库，完成 v0.1 设计文档和调研报告。'
    },
    {
      date:     '2026-W14',
      label:    '上周',
      layer:    'L3',
      fidelity: 0.77,
      content:  '项目启动，确定层级 Pooling 核心思路，完成相关工作调研。'
    }
  ],
  total_tokens: 480
}
```

**注入结构示意**（application 自己决定怎么拼）:
```
=== 2026-04-06（昨天）· 保真度 0.88 ===
上午：与团队确定 compress endpoint...

=== 2026-04-05 · 保真度 0.88 ===
建立仓库，完成 v0.1...

=== 2026-W14（上周）· 保真度 0.77 ===
项目启动...
```

**关键点**: 内容与当前对话话题无关，纯粹按时间近度返回。越近 token 越多，越远 token 越少，天数超过阈值后自动切换为周粒度。

---

### `POST /recall`

**调用方**: agent（通过 MCP tool 主动发起）  
**作用**: 语义检索，agent 意识到需要某段记忆时使用

```typescript
Request: {
  query:        string
  max_tokens?:  number   // 默认 400
  min_fidelity?: number  // 过滤保真度过低的记忆
}

Response: {
  chunks: [
    {
      memory_id:    string
      content:      string
      layer:        'L1' | 'L2' | 'L3' | 'L4' | 'L5'
      fidelity:     number
      timestamp:    string
      distance_days: number
    }
  ]
}
```

**关键点**: DoraMemory 只返回数据，不注入 prompt。agent 拿到结果后自己决定怎么用。

---

### `POST /remember`

**调用方**: agent（通过 MCP tool 主动发起）  
**作用**: 强化某条记忆 —— 调高重要程度，或修正压缩层的内容

```typescript
Request: {
  memory_id:   string
  importance?: number   // 0~1，所有层级均可设置
                        // 超过阈值（默认 0.9）触发 flashbulb 保护，永不压缩
  content?:    string   // 仅 L2+ 可用，L1 传入直接报错
                        // 人工修正摘要内容
}

Response: {
  updated:        MemoryNode
  human_verified: boolean   // content 被人工编辑后为 true
  fidelity:       number    // human_verified=true 时 fidelity 会上调
}
```

**L1 限制理由**: 原始内容是 ground truth，不允许修改，只能调整重要程度。  
**L2+ 编辑后 fidelity**: 人工校正比自动摘要更可靠，fidelity 重置为 `min(original × 1.2, 1.0)`。

## Fidelity 衰减模型

```
fidelity(n) = 1.0 × α^n

n = 压缩次数（L1 写入时 n=0，每经过一层压缩 n+1）
α = 压缩系数（用户配置）
```

**不同 α 下各层保真度**:

```
层级   n    α=0.95  α=0.88  α=0.70
──────────────────────────────────
L1     0    1.00    1.00    1.00
L2     1    0.95    0.88    0.70
L3     2    0.90    0.77    0.49
L4     3    0.86    0.68    0.34
L5     4    0.81    0.60    0.24
```

**α 与模型能力的关系**: 模型越强，摘要保留信息越多，α 越高。

```typescript
// 配置示例
{
  compression_model: "claude-sonnet-4-5",
  compression_coefficient: 0.88   // 可覆盖，否则用模型预设
}
```

**Flashbulb Memory**: importance >= 阈值的记忆永不参与压缩，n 始终为 0，fidelity 永远为 1.0。

## 调用方分工

```
agent 框架（自动，无感知）     agent 本身（主动，MCP tool）
────────────────────────       ──────────────────────────
POST /compress                 POST /recall
GET  /memory                   POST /remember
```

---

## 离线压缩（内部，不暴露接口）

```
/compress 写入 → L1（fidelity=1，不加工）
                      ↓ cron（每日）
                     L2（日摘要，α^1）
                      ↓ cron（每周）
                     L3（周摘要，α^2）
                      ↓ cron（每月）
                     L4（月摘要，α^3）
                      ↓ cron（每年）
                     L5（年骨架，α^4）
```

写入链路和压缩链路完全解耦，互不阻塞。

## Decision Log

### DDR-004: 放弃 proxy 方案
**决策**: 不做 LLM API 代理，不修改用户原始 prompt。  
**理由**: 透明度问题，调试困难，企业场景合规风险。  
**日期**: 2026-04-07  
**状态**: Accepted

### DDR-005: compress endpoint 作为写入唯一触发点
**决策**: 记忆写入绑定到 agent 框架的 context 压缩调用。  
**理由**: 该调用本已存在，零额外成本；调用方完全知情；不侵入 LLM 调用链。  
**日期**: 2026-04-07  
**状态**: Accepted

### DDR-006: injection 纯时间驱动
**决策**: /memory 返回内容与当前对话话题无关，只按时间近度排列。  
**理由**: 语义相关的记忆由 agent 主动 recall；自动注入应该像"近期背景"而非"语义检索结果"。  
**日期**: 2026-04-07  
**状态**: Accepted

### DDR-007: fidelity 衰减公式
**决策**: fidelity(n) = α^n，α 用户可配置，与压缩模型能力强相关。  
**理由**: 简单可解释，参数语义明确，便于用户根据实际模型调整。  
**日期**: 2026-04-07  
**状态**: Accepted

### DDR-008: L1 内容不可编辑
**决策**: /remember 对 L1 只允许修改 importance，不允许修改 content。  
**理由**: 原始内容是 ground truth，是整个 fidelity 体系的基础。  
**日期**: 2026-04-07  
**状态**: Accepted
